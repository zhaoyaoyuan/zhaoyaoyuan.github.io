IO流 文件传输

  1. IO流概述和分类

    1. IO流概述

      1. IO:输入输出(Input/Output)
      2. 流:是一种抽象概念,是对数据传输的总称,也就是说在设备间的传输成为流,流的本质是数据传输
      3. IO流就是用来处理设备间数据传输问题的常见的应用:文件复制;文件上传;文件下载

    2. IO流分类

      1. 按照数据的流向

        1. 输入流:读数据 fis.read() (读取文件数据)
        2. 输出流:写数据 fos.write()(将数据写入文件)

      2. 按照数据类型分

        1. 字节流(byte)

          ​ byte[] bys = new byte[1024];

          ​ int len ;

          ​ while((len = fis.read(bys))!=-1){

          ​ fos.write(bys,0,len)

          ​ }

          ​ 字节输出流;字节输入流

        2. 字符流(char)

          1. char[] chs = new char[1024];

          ​ int len ;

          ​ while((len = fis.read(chs))!=-1){

          ​ fos.write(chs,0,len)

          ​ }

          1. String s;

          ​ while((s=fis.read())!=null){

          ​ fos.write(s)

          ​ }

          ​ 字符输入流;字符输出流

        一般来说,是按照数据类型分类的

        1. 记事本打开,能看懂:用字符流

          ​ 看不懂:用字节流

          ​ 不知道:用字节流

  2. 对象序列化流

    1. 定义:就是将对象保存到磁盘中,或者在网络中传输对象。这种机制就是使用一个字节序列表示一个对象,该字节序列包含:对象的类型、对象的数据和对象中存储的属性等信息；字节序列写到文件之后相当于文件中持久保存了一个对象的信息，反之，该字节还可以从文件中读取回来，重构对象，对它进行反向序列化

    2. 要实现序列化和反序列化就要使用对象序列化流和对象反序列化流

      1. 对象序列化流：ObjcetOutputStream
      2. 对象反序列化流：ObjcetInputStream

    3. 对象序列化流的三个问题

      1. 用对象序列化流序列化了一个对象后，假如我们修改了对象所属的类文件，读取数据会不会出现问题?

        会出问题,会抛出InvalidClassException异常

      2. 如果出问题了,如何解决?

        给对象所属的类加一个serialVersionUID

        private static final long serialVersionUID = 42L

      3. 如果一个对象的某个成员变量的值不想被序列化,又该如何实现?

        给该成员变量加transient的关键字修饰,该关键字标记的成员变量不参与序列化过程

    4. 对象反序列化流:ObjcetInputStream

      2. 构造方法:

        ObjcetInputStream(inputStream in):创建从指定的inputStream读取的ObjcetInputStream

      3. 反序列化对象的方法
