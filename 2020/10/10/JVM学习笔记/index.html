    类加载子系统

    1. 作用

      ​ 类加载器将类文件加载到虚拟机的内存

    2. 类加载过程

      1. 加载:在硬盘上查找并通过IO流写入字节码文件,(会将源代码编译打包成一个jar包)
      2. 链接:执行下面三个步骤
        1. 验证:校验字节码文件的正确性
        2. 准备:给类的静态变量分配内存,并赋予默认值 0,null等
        3. 解析:将符号引用替换为直接引用,该阶段会将静态方法替换为指向数据所存内存的指针或者句柄.这是所谓的静态链接过程(类加载阶段完成);动态链接是程序运行期间完成的符合引用替换为直接引用.
          1. 符号引用:class文件中,将一些单词,方法等转化为#num的格式代表
          2. 直接引用:当加载到内存中时,会将符合引用转化为地址,同一份符号引用在不同的虚拟机加载时转化成的直接引用可能是不同的
      3. 初始化:对类的静态变量初始化为指定的值(自己赋的值),执行静态代码块
      4. 使用:将class传入到运行区
      5. 卸载:运行完毕.

    3. 类加载器

      1. 启动类加载器:加载核心类库 bin目录下
      2. 扩展类加载器:加载扩展目录中的jar包 斌目录下的ext目录下的jar包
      3. 应用程序类加载器:自己写的class文件
      4. 自定义加载器:用户自定义路径下的jar包

    4. 双亲委派机制

      ​ 当加载问件时,应用程序类不会直接加载文件;

      应用程序类–(委托)–>扩展类加载器加载文件–(委托)–>启动类加载器,

      当启动类加载器找不到问件时,会打回请求给扩展类加载器,当扩展类加载器找不到文件时,会打回请求给应用程序类加载器加载文件。

    5. 双亲委派机制优点

      1. 沙箱安全机制:防止核心类库被篡改
      2. 避免类的重复加载:保证被加载类的唯一性

    运行时数据区

      2. 存储.class文件,包括里面的常量,静态变量,类元信息的。方法去的空间属于直接内存,不是JVM分配的内存。

        1. 局部变量:存储局部变量
        2. 操作数栈:存储临时数据
        3. 动态链接
        4. 方法出口:将方法的返回值返回到main栈帧里的局部变量里
        5. 操作数栈十数据的临时中转站,当给变量赋值时或者参与运算时,都会先将数据存储到操作数栈,然后在进行赋值或者参与运算。参与运算的到的结构再次进入到操作数栈,然后在赋值给局部变量里的变量.

      4. 本地方法栈

        1. 存储底层方法,C语言写的房发直接与 底层交互
        2. 当使用时.也会加载到线程中

      5. 堆内存

        2. 存储创建出来的对象
        3. GC的工作区.如上图所示
          1. 垃圾处理流程如下
            1. Eden满触发minorGC

      6. 逃逸分析

        1. JVM三种运行模式
          1. 解释模式:执行一行字节码,就编译一次机器码
          2. 编译模式:先将字节码全部编译成为机器码,然后依次执行完毕
          3. 混合模式:依然使用解释模式执行,但是对于一些”热点”代码采用编译模式执行,JVM一般采用混合模式执行
        2. 对象逃逸分析
          1. 就是分析对象作用域
          2. 当对象在一个方法中创建后,可能是作为参数被其他方法调用,或者作为返回值被返回出去。
          3. 当对象明确只使用一次,且栈内存空间足够时,JVM会将对象创建在栈内存中,随着方法出栈,减少GC次数

    2. JVM内存分配和回收策略

      1. 小容量对象有一个年龄计数器，存活15次之后会将其放入老年代
      2. 大容量对象如果新生代可以容纳，则先放入新生代，当放不下之后将其放入老年代（可以设置将超过指定大小的对象直接放老年代）
      3. 动态年龄分配：当minorGC n次时已经将年轻代一般空键占满，则直接将之后产生的对象放入老年代。

    3. 判断对象是否可以被回收

      1. 引用计数法(已过时)
      2. 可达性分析算法 寻找GCroot并将链接的对象标记为有用对象,找不到的GCroot的对象则为无用对象GCroot根一般为栈 ,本地方法栈以及方法区中
      3. 有时有引用的对象也可能会被回收
        1. 强引用:普通的对象
        2. 软引用:被SoftReference软引用类型的对象包裹。一般不会被回收,当GC之后内存不足以存放新对象才会被回收
        3. 弱引用:被WeakReference弱引用类型的对象包裹。GC时会直接回收掉
        4. 虚引用:也成为了幽灵引用或者幻影引用.基本不会被使用
      4. finalize()方法判断对象最终是否存活
        1. 当对象没有与GCroots链接的引用链时,判断该对象是否被Finalize()方法覆盖标记,如果被此方法标记,则只要重新与任意对象的引用链链接即可
        2. 否则就会被回收

    4. 判断类对象是否可以被回收

      1. 类的对象实例都已经被回收
      2. 类的加载器被回收
      3. 在其他地方没有类的引用,不能同过反射构建类的对象实例
      4. 上面三个条件必须全部满足,这个类才能被回收
